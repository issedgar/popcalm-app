# ACCESSIBILITY.md

## 1. Purpose

This document defines the accessibility rules the agent must follow when building interfaces with the **AI Development Framework (ADF)**.

Accessibility is not optional. It must be integrated into design, HTML, components, interactions, colors, and keyboard navigation.

## 2. Principles

The agent must ensure:

1. Semantic HTML.
2. Correct heading hierarchy.
3. Keyboard navigation.
4. Visible focus states.
5. Sufficient contrast.
6. Useful alternative text.
7. Real labels in forms.
8. ARIA only when needed.
9. Respectful animations.
10. Interactions understandable without relying only on color or hover.

## 3. Semantics

Use semantic elements when appropriate:

```html
<header>
<nav>
<main>
<section>
<article>
<footer>
```

Avoid using `div` for everything.

Each page must have:

- One single `h1`.
- Headings in logical order.
- Sections with clear purpose.

## 4. Links and buttons

Rules:

- Use `<a>` for navigation.
- Use `<button>` for actions.
- Do not use buttons as links or links as buttons.
- Interactive icons must have `aria-label`.
- External links must use `target="_blank"` and `rel="noopener noreferrer"`.

## 5. Images

Every image must have `alt` text.

The `alt` text must describe the image function, not just repeat the file name.

Correct example:

```txt
Preview of the Angular 22 Feature Showcase application
```

If an image is decorative, it may use `alt=""`, but only if it truly communicates no information.

## 6. Forms

When forms exist:

- Every field must have an associated `label`.
- Errors must be clear.
- Errors must be readable by assistive technologies.
- Required fields must be indicated visually and in text.
- Do not rely only on placeholders.

## 7. Keyboard

The agent must ensure that:

- All interactive elements are reachable by keyboard.
- Tab order is logical.
- Focus is visible.
- There are no keyboard traps.
- Menus, dropdowns, and tooltips work correctly.

## 8. Color and contrast

The design must meet minimum contrast:

- Normal text: 4.5:1.
- Large text: 3:1.
- Interactive elements: clearly identifiable.

Do not communicate state only through color.

## 9. Motion

Respect:

```css
@media (prefers-reduced-motion: reduce)
```

If the user prefers reduced motion:

- Reduce or remove animations.
- Avoid long transitions.
- Avoid decorative motion.

## 10. shadcn/ui and Radix

When using shadcn/ui with Radix:

- Keep accessible primitives intact.
- Do not remove ARIA attributes generated by components.
- Do not break focus management.
- Do not wrap interactive components in invalid ways.

## 11. Final checklist

- [ ] There is one single `h1` per page.
- [ ] Heading hierarchy is correct.
- [ ] Semantic elements are used.
- [ ] Links and buttons have correct roles.
- [ ] Interactive icons have `aria-label`.
- [ ] Images have useful `alt` text.
- [ ] Contrast is sufficient.
- [ ] Visible focus exists.
- [ ] Keyboard navigation works.
- [ ] Animations respect `prefers-reduced-motion`.
- [ ] External links are safe.
- [ ] State is not communicated only through color.
